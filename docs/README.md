## üìù Documentation Standards

### When to Add Documentation

- **New Features**: Add design notes to `/design/`
- **Architecture Changes**: Update audit docs if significant
- **Process Changes**: Update project management docs

### File Naming

- Use `UPPERCASE_WITH_UNDERSCORES.md` for major docs
- Use `lowercase-with-dashes.md` for minor/supporting docs
- Keep names descriptive but concise

### Organization

- Keep related docs together in subdirectories
- Update this README when adding new categories
- Keep the root directory clean (README.md + CONTRIBUTING.md only)
